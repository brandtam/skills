Agent skills · install walkthrough

/skills

A standalone set of agent skills. Pull them straight from GitHub, or add them as a Claude Code plugin. Pick a path below.

✓ Agent Skills spec ✓ GitHub-hosted ✓ Codex & Claude ✗ no npm ✗ no third-party installer

00 — What you're installing

A skill is just a folder.

Every skill in this repo is a self-contained folder under skills/ with a SKILL.md inside (plus any references/ or assets/ it needs). Your agent reads that folder and gains the workflow.

Because a skill is only files, there's nothing to compile and nothing to run from a registry. The two automated paths below just move these folders into the place your agent already looks — and you can always do it by hand.

Skills are read from ~/.agents/skills/ (Codex and other agents using that convention) or ~/.claude/skills/ (Claude Code's own folder). Claude Code can also consume this repo as a plugin, which is Path A.
Every skill here conforms to the open Agent Skills specification and is validated in CI with skills-ref. That's the standard the "copy into any agent" promise rests on — any compliant agent reads the same folder.

02 — Before you start

Short list.

ForYou need
Path AClaude Code installed and signed in. That's it — it fetches the repo itself.
Path B / Cgit, plus a Bash-capable shell (macOS/Linux ship one; on Windows use Git Bash or WSL).

Path A — Claude Code marketplace

Native, auto-updating, Claude only.

1

Register the marketplace

Run this inside Claude Code. It clones the repo from GitHub and reads its plugin catalog — no npm, no registry.

in Claude Code
/plugin marketplace add brandtam/skills
2

Install the plugin

The repo ships as a single plugin named brandt-skills in a marketplace named brandt. Installing it brings in all five skills at once.

in Claude Code
/plugin install brandt-skills@brandt
3

Confirm

Open the plugin menu to see it listed and enabled. The skills are now available to Claude by name (e.g. ask it to "write a commit" or type the skill's slash command).

in Claude Code
/plugin
Updating later: re-running /plugin marketplace add brandtam/skills refreshes the catalog; Claude Code pulls the latest from GitHub. To remove it, use the /plugin menu and uninstall brandt-skills.

Path B — git clone + install.sh

Cross-tool. Works for Codex and for Claude.

1

Clone the repo

A normal git clone. The skills are right there in skills/ for you to inspect before anything is installed.

terminal
git clone https://github.com/brandtam/skills.git
cd skills
2

Run the installer

install.sh copies skill folders into ~/.agents/skills/. It's plain Bash — it only reads the folders sitting next to it and copies them. Nothing is fetched or executed from the network.

install everything
./install.sh
or just the ones you want
./install.sh write-commit prepare-release
install into Claude's folder instead
DEST=~/.claude/skills ./install.sh
3

Reload your agent

Restart Codex / Claude (or reload skills) so it re-scans the skills directory. The new skills are now available.

Updating later: git pull inside the cloned folder, then run ./install.sh again. It replaces each skill folder cleanly.

Path C — by hand

No script. Just a copy.

If you'd rather not run a script at all, copy any single skill folder into your agent's skills directory. This is the most transparent option — you can read every file first.

terminal
# from inside the cloned repo, pick whichever skill you want
cp -R skills/write-commit ~/.agents/skills/

Swap ~/.agents/skills/ for ~/.claude/skills/ if you're feeding Claude Code directly. To remove a skill, delete its folder.

04 — What's in the box

Five skills.

SkillWhat it does
prepare-releaseCut a release from pending changesets — preview the next version, generate the changelog, archive consumed notes, stage tag / GitHub Release steps.
write-change-setWrite or update a changeset file for a PR or branch, scaffolding the changeset workflow if it's missing.
write-commitGenerate structured commit messages and optionally commit — honors repo commit templates, requires your approval before committing.
uatRun a user-acceptance-testing session: collect observations, grill for decisions, then generate a PRD.
resurrect-projectRevive a dormant project — survey the code as historical signal, interview on today's intent, capture decisions, scaffold a rewrite plan.
